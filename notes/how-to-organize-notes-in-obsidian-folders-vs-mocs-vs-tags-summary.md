# How to Organize Notes in Obsidian - Summary

This video compares three ways to organize notes in Obsidian: folders, MOCs, and tags. The main point is that each method has a different purpose, and the best system often combines them instead of treating one as the only correct answer.

## Main Problem

New Obsidian users often get stuck asking:

- Where should this note go?
- Which folder does it belong in?
- Should I use folders, tags, PARA, Johnny Decimal, or MOCs?

The speaker argues that the confusion happens because people use Obsidian for different reasons: research, journaling, projects, study, work, or personal knowledge management.

Instead of copying someone else's system, the goal is to understand how each method behaves over time.

## Folders

Folders are familiar because they work like file explorers. They are useful when parts of life or work need physical separation.

Examples where folders make sense:

- Templates
- Journal
- Home
- Study
- Work
- Content

Folders are good when the areas do not need many connections.

## Problems With Folders

Folders create friction when a note belongs in more than one place.

Example:

A note called `How to read file in Python` could belong under:

- `Python Basics`
- `Python Project 1`

If you copy it into both places, you break the single source of truth. If you put it in only one place, it may be hard to find from the other context.

Folders can also become deep rabbit holes. After months, you may forget where notes are, why they exist, or how to find them.

The speaker's folder rule:

```text
Never go more than two levels deep.
```

Use folders for high-level separation, but avoid deeply nested folder structures.

The speaker also recommends navigating with Obsidian's quick switcher instead of manually browsing folders:

- `Control + O`

## Folderless Organization

The second method is using no folders at all. All notes live together, and organization happens through MOCs.

MOC means Map of Content.

A MOC is an index note that links to related notes.

Examples:

- `MOC Investment`
- `MOC Python`
- `MOC Health`

If you want to learn about stocks, you open `MOC Investment`, then navigate to stock-related notes from there.

## Benefits Of MOCs

MOCs reduce the question of where a note should physically live. Instead, you ask which index note should link to it.

Benefits:

- Less decision fatigue
- Less folder digging
- Easier long-term scaling
- Notes can appear in multiple contexts
- Better for connected knowledge

The speaker suggests that beginners can start with a mostly MOC-based system and add folders later if needed.

## Attachments In A Folderless System

One issue with folderless organization is attachment clutter. Images and files can appear beside notes in the root folder.

The solution is to set Obsidian's attachment location to a specific folder, such as:

```text
attachments
```

This keeps media files separate while allowing notes to stay in the main directory.

## Tags

The speaker argues that tags should not be the main structure.

The key line is:

```text
Tags are not structure. Tags are signals.
```

Tags are best for filtering and discovery.

Examples:

- `#course`
- `#story`
- `#quote`
- `#task`

Clicking a tag shows all notes with that tag, similar to hashtags on social media.

Tags are useful for finding patterns across notes and discovering related material later, but they should not replace folders or MOCs.

## Recommended Hybrid Approach

The speaker's preferred approach combines methods:

- Use folders for high-level separation.
- Use MOCs to create connections and navigation paths.
- Use tags as signals for filtering and discovery.

For example, a vault might use folders at the top level for broad areas, then contain MOCs inside those areas for connected navigation.

## Main Takeaway

Folders, MOCs, and tags solve different problems:

- Folders create separation.
- MOCs create structure and connection.
- Tags create discoverability.

Use folders when physical separation makes sense, avoid deep nesting, rely on MOCs for connected knowledge, and use tags as lightweight signals rather than as the main organizing system.
