# Why use LCC Classifications?

Because naming is hard.

I decided not to invent a new classification system, but to use one that is open and free.

## Qualitative reasons

Using an existing classification system reduces the number of arbitrary decisions needed when creating notes. Instead of asking "what folder should this go in?" from scratch each time, the vault can ask "what broad shelf does this belong near?"

LCC also gives the vault a stable external vocabulary. The labels are not perfect, but they are durable, documented, and broadly understandable. That makes them useful as orientation markers for both humans and AI.

The classification does not need to be exact to be useful. A broad code can act as a retrieval cue, while MOCs and links handle the more personal and flexible navigation layer.

Other benefits:

- It avoids inventing a private taxonomy that has to be redesigned later.
- It gives related notes a visible neighborhood even when they are linked from many MOCs.
- It separates storage orientation from topic navigation.
- It makes new-note placement more repeatable.
- It gives AI agents a lookup process instead of relying only on intuition.
- It supports future cleanup because misplaced notes can be compared against a known classification spine.

The goal is not formal library cataloging. The goal is practical orientation.
